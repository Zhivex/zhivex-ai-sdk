# Security Policy

## Reporting a Vulnerability

Please do not open a public GitHub issue for suspected vulnerabilities.

Instead, report the issue privately to the maintainers with:

- a description of the vulnerability,
- affected packages and versions,
- reproduction steps or a proof of concept,
- any suggested mitigations, if available.

If GitHub Security Advisories are enabled for the repository, prefer using private vulnerability reporting there.

## Response Expectations

We will aim to:

- acknowledge the report promptly,
- validate the issue,
- work on a fix or mitigation,
- coordinate disclosure once users have had a reasonable chance to update.

## Scope

Please report vulnerabilities related to:

- published package code,
- build and release configuration,
- credential handling in repository automation,
- dependency issues with a direct impact on users of this SDK.

Reports that are purely support questions or feature requests should go through normal repository discussions or issues.
