Add the agent platform foundation layer across durable agent runs, advanced tool registries, safety policies, replay/evaluation, trace observability, provider parity, fixture/report workflows, and production subagent orchestration.

This release adds compatible durable run primitives with idempotency, cooperative cancellation, schema-versioned state, production run states (`waiting_approval`, `cancel_requested`, `timed_out`, and reserved `queued`), and agent run timeout policy; stable runner/session APIs with in-memory, file, SQLite, and Postgres `SessionService` implementations, session events, multi-turn context injection, resumable approval state, schema-versioned session records, v1 session migration helpers, file session pruning, and optional optimistic concurrency revisions with SQL compare-and-swap guards; beta declarative workflow APIs for sequential, parallel, and loop runner workflows, output keys, approval resume, dry workflow replay, workflow state persistence through `SessionService` fallback metadata or recommended dedicated schema-versioned `WorkflowStateService` stores, workflow artifact helpers, workflow evaluation/report helpers, and workflow evaluation diff reports; beta artifact service APIs with in-memory, file-backed, SQLite, and Postgres stores for schema-versioned session/workflow/agent run artifacts plus base64 attachment metadata, binary artifact storage, external artifact references for app-owned blobs, integrity verification, file-store cleanup/pruning, and documented production binary storage boundaries; beta migration helpers and compatibility fixtures that normalize legacy artifact, workflow run, and workflow state records to schema version 1; a beta `zhivex-ai` CLI for sessions, artifacts, artifact integrity/cleanup/pruning, workflow replay, workflow reports, local workflow execution, workflow artifact saving, first-class persisted workflow state inspection/pruning, legacy session workflow-state inspection, help output, and evaluation report comparison; a beta API stability manifest plus declaration snapshot guards that define the RC runtime/type boundary and guard against unclassified public exports; experimental advanced tool registry helpers with HTTP tools, permissions, audit metadata, registry inspection, and tool fixtures; stable safety policy helpers for approvals, redaction, and budget guards; stable replay, mock, evaluation, report, trace, cost, and latency helpers; stable provider parity matrix rendering and drift reports; and native subagent primitives with shared stores, budgets, cascade cancellation, hierarchical traces, multi-agent evaluations, fail-fast parallel agent groups, and shared defaults.

Promote Bedrock OpenAI-compatible Responses runtime to Tier A agent support with typed hosted tool helpers, remote MCP approval responses, and provider-data approval parsing. Add AWS-native Bedrock AgentCore MCP client and ToolSet helpers for SDK-managed remote tools on Converse/shared agent loops.

Promote DeepSeek to Tier B agent support for portable tool loops, tool choice, JSON object output, streaming, and documented thinking mode while keeping hosted tools, remote MCP, web search, embeddings, audio, and realtime capabilities explicitly unsupported.
